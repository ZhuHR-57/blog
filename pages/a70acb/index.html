# 理论基础

  1. 回溯算法是什么？

回溯是枚举，一种纯暴力的遍历

  1. 为什么还需要回溯呢？

因为有些场景能暴力出来就很好了，解决的 n 个 for 循环的情况

  1. 有哪些场景需要用到回溯呢？
  • 组合问题：N 个数里面按一定规则找出 k 个数的集合
  • 切割问题：一个字符串按一定规则有几种切割方式
  • 子集问题：一个 N 个数的集合里有多少符合条件的子集
  • 排列问题：N 个数按一定规则全排列，有几种排列方式
  • 棋盘问题：N 皇后，解数独等等
  1. 排列和组合的区别？
  • 组合：{1, 2} 和 {2, 1} 在组合上，就是 一个集合 ， 不强调顺序
  • 排列：{1, 2} 和 {2, 1} 就是 两个集合 了， 强调顺序
  1. 如何理解回溯法？

理解为一种树形结构。

回溯法解决的都是在集合中递归查找子集，集合的大小就构成了树的宽度，递归的深度，都构成的树的深度。

void backtracking(参数) {
    if (终止条件) {
        存放结果;
        return;
    }

    for (选择：本层集合中元素（树中节点孩子的数量就是集合的大小）) {
        处理节点;
        backtracking(路径，选择列表); // 递归
        回溯，撤销处理结果
    }
}

# 录后想法

回溯 = 递归 + for循环 =》 解决了【k 个 for 循环...】问题

回溯可以理解为需要 for循环 配合 递归 帮忙遍历一颗树

那么该如何实现 for循环+递归 呢？

  1. 建立宏观的整体思考过程
    1. for 循环：遍历节点中的元素
    2. 树枝： 遍历到的元素
    3. 递归： 负责控制 for 循环的个数

  1. 进入单层的 for 循环后，该从哪里开始呢、哪里结束呢？【也就是 for ( int i = ? ; i < ? ; ++i )】
    1. 这里的小技巧就是用指针标记： startIndex =》 int i = startIndex;
    2. startIndex 来自递归参数的传递
    3. 到总元素的个数 n 结束
// 结合到一起就是
for(int i = startIndex; i < n; ++i){ ... }
1
2
  1. 确定了循环的开始和终止，那么该如何 收集元素 呢？【也就是 for 循环内的具体操作】
    1. 保存树枝上的元素
    2. 继续处理延伸出去的树枝
    3. 删除已经处理的树枝
for(int i = startIndex; i < n; ++i) {
    
    // 保存树枝上的元素 
    // path：用于保存树枝的数组
    path.push_back( 元素 );
    
    // 继续处理延伸出去的树枝
    // 递归，传入下一个要处理的元素位置
    backtracking(... , 下一个要处理的元素位置);
    
    // 删除已经处理的树枝
    path.pop_back();

}

以下是整体的代码实现，已 AC (✔️)

class Solution {
private:
    vector<int> path;
    vector<vector<int>> result;
    // 回溯算法
    void backtracking(int n,int k,int startIndex){

        // 终止条件
        if(k == path.size()){
            // 收集结果
            result.push_back(path);
            return;
        }

        /*
            单层搜索
        */
        // 树形结构的每一层节点都是一层for循环
        // 都是从startIndex开始
        for(int i = startIndex; i <= n; ++i){
            // 收集路径上的元素
            path.push_back(i);
            // 递归
            backtracking(n,k,i + 1);
            // 弹出回溯
            path.pop_back();

        }

    }
public:
    vector<vector<int>> combine(int n, int k) {

        path.clear();
        result.clear();
        backtracking(n,k,1);

        return result;
    }
};

嘿嘿，没想到吧，还有没结束，还可以优化（ 剪枝操作 ）

理论上回溯的纯暴力 (全遍历) 的算法，那有些情况是不是可以用遍历到呢？

当然是有的，比如：

当 n = 4，k = 4 的话，

  • 第一层 for 循环，从元素 2 开始的遍历都没有意义了。
  • 第二层 for 循环，从元素 3 开始的遍历都没有意义了

那该如何处理呢？

  1. 已经选择的元素个数：path.size ();
  2. =》还需要的元素个数为: k - path.size ();
  3. =》在集合 n 中 最少 要从该起始位置 : n - (k - path.size ()) + 1，开始遍历

为什么有个 + 1 呢，因为包括起始位置，我们要是一个左闭的集合。

举个例子，n = 4，k = 3， 目前已经选取的元素为 0（path.size () = 0），n - (k - 0) + 1 即 4 - ( 3 - 0) + 1 = 2。

  • 从 2 开始搜索都是合理的，可以是组合 [2, 3, 4]

  • 往后从 3 开始就不够了三个数了

  • 往前从 1 开始当然也足够

理论知道了，那该如何修改代码呢？

这个就比较简单，只需要修改 for循环的结束位置 就行

for (int i = startIndex; i <= n - (k - path.size()) + 1; i++){ ... }
1

完整代码实现如下，已 AC (✔️)

class Solution {
private:
    vector<int> path;
    vector<vector<int>> result;
    // 回溯算法
    void backtracking(int n,int k,int startIndex){

        if(k == path.size()){
            result.push_back(path);
            return;
        }

        // 修改代码部分
        for(int i = startIndex; i <= n - (k - path.size()) + 1; ++i){
            path.push_back(i);
            backtracking(n,k,i + 1);
            path.pop_back();
        }

    }
public:
    vector<vector<int>> combine(int n, int k) {

        path.clear();
        result.clear();
        backtracking(n,k,1);

        return result;
    }
};
