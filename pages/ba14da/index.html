# 动态规划是什么？

  • 动态规划，英文：Dynamic Programming，简称 DP

  • 解决什么问题：如果某一问题有很多重叠子问题，使用动态规划是最有效的

  • 是怎么解决的：每一个状态一定是由上一个状态推导出来的

  • 与贪心的区别：贪心没有状态推导，而是从局部直接选最优的

更确切地说：

它与暴力的区别在哪里？我们的暴力枚举了 “使用的硬币”，然而这属于冗余信息。我们要的是答案，根本不关心这个答案是怎么凑出来的。譬如，要求出 f (15)，只需要知道 f (14),f (10),f (4) 的值。** 其他信息并不需要。** 我们舍弃了冗余信息。我们只记录了对解决问题有帮助的信息 ——f (n).

我们能这样干，取决于问题的性质：

求出 f (n)，只需要知道几个更小的 f (c)。我们将求解 f (c) 称作求解 f (n) 的 “子问题”。

这就是 DP（动态规划，dynamic programming）.

将一个问题拆成几个子问题，分别求解这些子问题，即可推断出大问题的解。

# 录前想法

这题目很经典，以前做过

直接 AC，去看卡哥的分析

class Solution {
public:
    int fib(int n) {

        if(n <= 1) return n;

        int pre = 0;
        int cur = 1;
        int tmp;

        while(--n){
            tmp = cur;
            cur = pre + cur;
            pre = tmp;
        }

        return cur;
    }
};

# 录后想法

  1. 确定 dp 数组（dp table）以及下标的含义： dp[i] ， i 指的是第几个斐波那契数

  2. 确定递推公式： dp[i] = dp[i-1] + dp[i-2]

  3. dp 数组如何初始化： dp[0] = 0 dp[1] = 1

  4. 确定遍历顺序：顺序遍历 for(int i = 2; i < n; ++i)

  5. 举例推导 dp 数组（Debug）

根据上面的分析，实现代码如下：

class Solution {
public:
    int fib(int n) {

        if(n <= 1) return n;
        int dp[n+1];

        dp[0] = 0;
        dp[1] = 1;

        for(int i = 2; i <= n; ++i){
            dp[i] = dp[i-1]+dp[i-2];
        }

        return dp[n];
    }
};

# 录前想法

写了几个实例后发现，就是一道求斐波那契数的题目

class Solution {
public:
    int climbStairs(int n) {

        // 1. 确定dp数组（dp table）以及下标的含义
        int dp[n+1];
        // 2. dp数组如何初始化
        dp[0] = 1,dp[1] = 1;
        // 3. 确定遍历顺序
        for(int i = 2;i <= n; ++i){
             // 4. 确定递推公式
            dp[i] = dp[i-1]+dp[i-2];
        }
        return dp[n];
    }

    // 1：1
    // 2: 1+1 2
    // 3: 1+1+1 1+2 2+1
    // 4: 1+1+1+1 2+1+1 1+2+1 1+1+2 2+2
    // 5: 1+1+!+1+1 1+1+1+2 1+2+2 2+1+2 2+2+1 
};

# 录后想法

自己写时只是从数的角度，感觉出是斐波那契数列，卡哥这里说明白了为什么是

根本原因就是：一次只能走 两步

举个例子：

n = 1 ：1

n = 2 ：2

n = 3 ，3 只能要么是从 1 走上来，要么是从 2 走上来

题目问有几种方法：也就是从 1 走上来的方法 + 从 2 走上来的方法 => 斐波那契数

# 录后想法

  1. 确定 dp 数组（dp table）以及下标的含义

dp[i] 中 i 为第几阶楼梯， dp[i] 为到达第几阶楼梯的最小消费

  1. 确定递推公式

dp[i] = min(dp[i-1]+cost[i-1],dp[i-2]+cost[i-2]);

为什么是 dp[i-1] + cost[i-1] ，不是 dp[i-1] 呢？

因为最开始 (到达第一阶梯) 时是 0【 dp[i-1] 】，出发到下一个阶梯时才消耗本阶梯的费用【 cost[i-1] 】

选取一个最小值

  1. dp 数组如何初始化

一开始站在台阶上，只要不走，就都是 0

ap[0] = 0，dp[1] = 0

  1. 确定遍历顺序

顺序遍历

根据以上分析，实现代码如下，已 AC (✔️)

class Solution {
public:
    int minCostClimbingStairs(vector<int>& cost) {

        // 1. 确定dp数组（dp table）以及下标的含义
        int dp[cost.size() + 1];
        // 2. dp数组如何初始化 
        dp[0] = 0,dp[1] = 0;
        // 3. 确定遍历顺序
        for(int i = 2;i <= cost.size(); ++i){
            // 4. 确定递推公式
            dp[i] = min(dp[i-1]+cost[i-1],dp[i-2]+cost[i-2]);
        }

        return dp[cost.size()];
    }
};
